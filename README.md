# Silent Pass（Vibe Coding）

Silent Pass 是一个自用 Android 应用，用来解决“手机处于静音或震动时，指定联系人来电仍然响铃”的场景。应用不做来电拦截、不拒接电话，只在选中的联系人来电时尝试把系统铃声模式切换为响铃。

## 应用信息

- 应用名称：Silent Pass
- 包名：`com.islate.silentpass`
- 图标：白底蓝色 `sp` 几何字标。
- 技术栈：Kotlin、Jetpack Compose、Material 3、`CallScreeningService`、SharedPreferences。

## 当前功能

### 1. 手动选择联系人

- 应用不会默认读取全部通讯录。
- 只有点击页面里的“选择”按钮时，才会打开系统联系人选择器。
- 选择联系人后，应用读取该联系人的电话号码并加入应用内列表。
- 如果同一个联系人被重复选择，会用新读取到的信息覆盖旧条目。
- 联系人列表中可以删除已选择的联系人。

### 2. 联系人响铃开关

- 每个已选择联系人右侧都有一个开关。
- 开启后，该联系人的号码会进入“允许静音时响铃”的本地快照。
- 关闭后，该联系人仍保留在列表中，但来电不会触发静音转响铃。
- 添加、删除或切换开关后，需要点击“保存设置”才会写入本地快照。

### 3. 本地号码快照

- 点击“保存设置”时，应用会保存当前已开启联系人的号码快照。
- 来电时只匹配本地保存的号码快照，不读取通讯录。
- 如果后续修改了联系人号码，需要重新选择该联系人并再次保存。
- 这种设计减少了来电瞬间的后台工作，也减少了对通讯录的依赖。

### 4. 来电时临时切换响铃

- 当保存快照中的联系人来电时，应用会检查当前手机铃声模式。
- 如果手机已经是响铃模式，应用不做任何额外操作。
- 如果手机是静音或震动模式，应用会尝试切换到响铃模式。
- 应用不会自动恢复静音或震动，后续由用户手动调整。

### 5. 不做来电拦截

- 应用使用 `CallScreeningService` 作为来电到达时的系统触发点。
- 当前逻辑始终允许来电通过。
- 不会拒接电话。
- 不会隐藏来电通知。
- 不会跳过通话记录。

## 权限说明

### 联系人权限

用途：

- 手动选择联系人后读取该联系人的电话号码。

特点：

- 不会在来电时读取通讯录。
- 不会扫描全部通讯录。
- 保存快照后，即使不打开联系人选择器，来电匹配也只使用本地保存的号码。

### 来电检测角色

用途：

- 让系统在来电时调用应用的 `CallScreeningService`。

说明：

- 应用必须被设置为系统的来电检测/来电筛选应用，才能在来电时执行响铃逻辑。
- 当前应用虽然使用来电筛选服务，但不执行拦截行为。

### 勿扰与响铃控制权限

用途：

- Android 在静音、震动、勿扰相关场景下限制普通应用修改铃声模式。
- 授予该权限后，应用才能在符合条件的来电到达时切换到响铃模式。

说明：

- 如果没有该权限，匹配到联系人来电时也可能无法切换为响铃。
- 小米/HyperOS 可能还会有额外的后台和省电策略限制。

## 使用流程

1. 安装并打开应用。
2. 在“权限状态”区域依次确认：
   - 联系人：已启用
   - 来电检测：已启用
   - 静音响铃：已启用
3. 在“联系人”卡片中点击“选择”。
4. 从系统联系人选择器中选择一个联系人。
5. 确认该联系人右侧开关已开启。
6. 点击“保存设置”。
7. 将手机调成静音或震动。
8. 使用该联系人的号码拨入测试。

## 省电策略

当前版本已经做了这些省电取舍：

- 不常驻后台。
- 不轮询。
- 不监听通讯录变化。
- 不在来电时读取通讯录。
- 不读取全部联系人。
- 不设置自动恢复静音定时器。
- 来电时只读取本地 SharedPreferences 中保存的号码快照。

因此，日常耗电主要来自系统在来电时短暂拉起 `CallScreeningService`。如果没有来电，应用基本没有持续后台工作。

## 小米手机建议

在小米 15 / HyperOS 上，如果功能不稳定，可以检查：

- 应用是否仍是来电检测/筛选应用。
- 勿扰权限是否仍然允许。
- 联系人权限是否允许。
- 如果系统省电策略导致来电时服务不触发，再考虑给应用设置更宽松的后台策略。

当前版本不需要常驻后台，建议先使用系统默认省电策略测试。

## 构建

Debug 构建：

```bash
./gradlew :app:assembleDebug
```

Release 构建：

```bash
./gradlew :app:assembleRelease
```

Release 包路径：

```text
app/build/outputs/apk/release/app-release.apk
```

当前 release 使用 debug 签名，便于自用安装；同时开启了 R8 压缩和资源压缩，以减小安装包体积。

## 安装

Debug 安装：

```bash
adb install -r app/build/outputs/apk/debug/app-debug.apk
adb shell am start -n com.islate.silentpass/.MainActivity
```

Release 安装：

```bash
adb install -r app/build/outputs/apk/release/app-release.apk
adb shell am start -n com.islate.silentpass/.MainActivity
```

## 重要限制

- 应用无法绕过系统或厂商对铃声模式的限制。
- 如果系统处于强勿扰、专注模式或厂商策略限制，切换响铃可能失败。
- 来电后应用不会自动恢复静音，需要用户手动调整。
- 修改联系人号码后，需要重新选择联系人并保存设置。
